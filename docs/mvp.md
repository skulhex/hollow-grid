# MVP Hollow Grid

Документ фиксирует ближайшую минимальную версию правил. Это не финальный дизайн, а рабочая опора для клиента, сервера и первых плейтестов.

## Идея

Hollow Grid — абстрактная пошаговая multiplayer strategy game про построение, защиту и разрыв сети на hex-grid.

Игроки расширяют активную сеть от своего `Core`, используют её для контроля пространства, отключают узлы соперника и готовят осаду вражеского ядра.

## Игровое ядро

- Матч рассчитан на 1-2 игроков.
- Основной режим — 2 игрока через WebSocket-сервер.
- Для ранней отладки допускается локальный режим с заглушкой второго игрока.
- Игроки ходят по очереди.
- За ход игрок может выполнить несколько действий в рамках доступных лимитов.
- Автоматические эффекты применяются в start-of-turn upkeep активного игрока.
- Сервер хранит и проверяет состояние матча.

## Поле и объекты

- Поле — фиксированный hex-grid.
- Рекомендуемый стартовый размер: радиус 3, всего 37 клеток.
- Координаты клеток хранятся в axial-системе: `q`, `r`.
- У каждого игрока есть один `Core` на своей стороне поля.
- `Core` имеет `5 HP`.
- `Node` — базовый узел сети, который игрок ставит во время хода. У каждого `Node` есть роль.
- Новые `Node` создаются как `Conduit`.
- `Conduit` проводит активную сеть и даёт influence на `control point`, но не даёт `Resource` и не наносит урон `Core`.
- `Harvester` — улучшенный свой активный `Conduit`. Он проводит сеть, даёт influence и может приносить `Resource`.
- `Striker` — улучшенный свой активный `Conduit`. Он проводит сеть, даёт influence и будет использоваться для атак после отдельного уточнения attack-правил.
- Центральная клетка — `control point`; на неё нельзя ставить `Node`.

## Правила MVP

- На одной клетке может быть только один объект.
- `Core` не двигается и не удаляется.
- Игрок побеждает, когда HP вражеского `Core` становится `0`.
- `Node` можно поставить только на свободную клетку рядом со своей активной сетью.
- Активная сеть — это `Core` и все `Node`, связанные с ним цепочкой соседних объектов игрока.
- `Node`, отрезанный от `Core`, становится неактивным.
- Неактивный `Node` остаётся на поле, но от него нельзя строить сеть.
- Отключённый `Node` остаётся на поле, блокирует постановку новых `Node`, не даёт influence и не считается частью активной сети.
- Отключённый `Node` сохраняет свою роль, но не даёт influence, `Resource` или урон.
- `Control point` контролируется влиянием вокруг него, а не объектом на самой клетке.
- Игрок контролирует `control point`, если рядом с ним больше активных объектов игрока, чем активных объектов соперника. Активные `Conduit`, `Harvester`, `Striker` и `Core` дают influence.
- При равенстве influence или отсутствии активного influence `control point` не контролирует никто.
- `Control point` не даёт победные очки. В MVP он создаёт позицию для `Harvester`, который даёт ресурсное преимущество в start-of-turn upkeep.

## Ресурсы и действия

- Игроки начинают с `1 Resource`.
- У игроков есть `Resource` для долгосрочного преимущества и будущих типов `Node`.
- В начале своего хода игрок получает лимиты действий:
  - `connection_actions_left = 1`;
  - `repair_actions_left = 1`.
- В начале своего хода все role-`Node` игрока восстанавливают `action_charges = 1`.
- За ход можно выполнить несколько действий, пока хватает соответствующих лимитов и `Resource`.
- Доступные действия:
  - `place_node` — поставить свой `Node`;
  - `repair_node` — восстановить свой отключённый `Node`;
  - `upgrade_harvester` — улучшить свой активный `Conduit` в `Harvester`;
  - `upgrade_striker` — улучшить свой активный `Conduit` в `Striker`;
  - `skip` — завершить ход.
- `place_node` тратит `1 connection action`.
- `repair_node` тратит `1 repair action`, доступен только для своего отключённого `Node` и не требует активного соседа.
- `upgrade_harvester` стоит `1 Resource`, не тратит лимиты действий и доступен только для своего активного `Conduit`.
- `upgrade_striker` стоит `1 Resource`, не тратит лимиты действий и доступен только для своего активного `Conduit`.
- `break_node` и `clear_node` будут уточнены отдельно и пока не входят в player-facing набор команд.
- Ход передаётся как единая `Action`-модель, пригодная для отправки на сервер:
  - `type` — одно из `place_node`, `repair_node`, `upgrade_harvester`, `upgrade_striker`, `skip`;
  - `player` — игрок, который отправляет действие;
  - `cell` — целевая клетка в axial-координатах `{ q, r }`; обязательна для действий с клеткой, отсутствует для `skip`.
- Пример payload: `{ "type": "place_node", "player": "player_1", "cell": { "q": -2, "r": 0 } }`.

## Ход и upkeep

Раунд считается парой завершённых ходов обоих игроков. Отдельной общей `Resolve phase` в текущем MVP нет.

1. `Start-of-turn upkeep` — активному игроку восстанавливаются лимиты, начисляется `Resource` от его активных `Harvester`, восстанавливаются заряды role-`Node`.
2. `Player actions` — игрок выполняет доступные действия и завершает ход через `skip`.
3. `Next player` — ход передаётся сопернику; после завершения ходов обоих игроков увеличивается номер раунда.

Минимальный start-of-turn upkeep:

- пересчитать активную сеть;
- восстановить `connection_actions_left` и `repair_actions_left`;
- начислить `Resource` активному игроку, если у него есть активный `Harvester` рядом с `control point`;
- восстановить `action_charges` role-`Node` активного игрока.

## Resource от Harvester

- `Resource` начисляется в start-of-turn upkeep активного игрока.
- Если у активного игрока есть хотя бы один активный `Harvester` рядом с `control point`, он получает `+1 Resource`.
- Несколько `Harvester` одного игрока рядом с `control point` пока не складываются: максимум `+1 Resource` за upkeep.

## Урон по Core

- Attack-правила для `Striker` будут уточнены отдельно.
- `Conduit` и `Harvester` не наносят урон `Core`.

## Визуальный стиль

- Минималистичное hex-поле.
- Тёмный или нейтральный фон.
- Разные цвета сетей для двух игроков.
- HUD отображает `Core HP`, оставшиеся лимиты действий и `Resource`.
- Активные `Node` отображаются ярче.
- Неактивные `Node` отображаются приглушённо.
- Отключённые `Node` отображаются отдельным повреждённым состоянием.
- `Conduit` использует обычный вид `Node`.
- `Harvester` и `Striker` отличаются простыми внутренними знаками ресурсного и warning цвета.
- Связи между активными соседними объектами можно показывать линиями или подсветкой рёбер.
- `Control point` визуально читается как нейтральный источник преимущества, а не финальная цель матча.
- Активный `Harvester`, который приносит `Resource`, визуально связывается с `control point`.

## Не входит в MVP

- аккаунты и авторизация;
- матчмейкинг;
- сохранение матчей в базе данных;
- рейтинг;
- чат;
- разные карты;
- ручные способности `Node`;
- `Defender` и `Hacker`;
- модули и здания;
- финальные attack-правила;
- случайные события;
- туман войны;
- сложные анимации;
- production-ready инфраструктура.

## Готово, когда

- Godot-клиент отображает hex-grid, `Core`, `Node` и `control point`.
- Игрок может выбрать действие и клетку.
- Клиент различает активные, неактивные и отключённые `Node`.
- Клиент отображает `Core HP`, оставшиеся лимиты действий и `Resource`.
- Сервер валидирует действия и пересчитывает связность сети.
- Ход начинается со start-of-turn upkeep.
- `Control point` даёт ресурсное преимущество через `Harvester`, но не победные очки.
- `Harvester` рядом с `control point` даёт `Resource` в начале хода владельца.
- Победа определяется разрушением вражеского `Core`.
- Два клиента могут подключиться к одному матчу.
- Проект можно запустить локально по понятной инструкции.
